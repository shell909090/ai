# 第二章：压缩测试

本章测试模型的对话压缩能力：将多轮对话历史浓缩为摘要，评估压缩比、关键信息保留率和输出语言稳定性。

---

## 语料构造

测试使用两类语料，均由问答对话构成：

### 2.A 英文语料（les_mis）

以《悲惨世界》为主题，5 轮英文问答，约 37K chars。关键词 8 个（人名、事件、情节要素）。

**构造方式**：通过 `2.0-compress_gen_baseline.py` 向 primary backend 提问，取真实模型回答作为语料，确保内容真实、语言地道。

### 2.B 中文语料（hong_lou_meng）

以《红楼梦》为主题，5 轮中文问答，约 12K chars。关键词 8 个。

**构造方式**：同上，中文提问取中文回答。

**生成命令**：

```bash
make eval/compress_baseline.jsonl
```

## 2.1 压缩质量评估

**脚本**：`2.1-compress_quality.py`（即现有 `2.1-compress_quality.py`）

**目的**：评估不同语言的压缩提示词对压缩质量的影响——压缩比、关键词命中率、输出语言稳定性。

**设计**：9 种压缩提示词变体（EN/ZH/JA × p1/p2/p3）× 2 个语料 × 3 次重复 = **54 次调用**。

**9 种提示词变体**：

| 语言 | p1（通用摘要） | p2（换词摘要） | p3（明确要求输出语言） |
|------|--------------|--------------|---------------------|
| EN | Please summarize... | Provide a comprehensive summary... | Please summarize IN ENGLISH... |
| ZH | 请总结以下对话... | 请对以下对话进行概括... | 请用中文总结... |
| JA | 以下の会話を要約してください... | 以下の会話の主要な内容を... | 以下の会話を日本語で要約してください... |

**评判指标**：

| 指标 | 定义 |
|------|------|
| `ratio` | output_tokens / input_tokens，越小越好 |
| `kw_pass` | 3 次中关键词命中 ≥ 50% 的次数（X/3） |
| `lang_dist` | 3 次输出语言分布，如 "en:2 zh:1" |

**通过条件**（单次）：finish_reason=completed 且 output 非空且关键词命中 ≥ 50%。

**运行**：

```bash
make eval/compress_results.csv
# 或
make prompt-ch2  # （待 Makefile 更新后）
```
